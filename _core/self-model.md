## Constraints

**Hard constraints:**
- **Budget:** No single subscription as a single point of failure. GitHub free,
  MkDocs free. Markdown files are the asset, not the platform.
- **Time:** System must fit around family and client work. Nothing requiring
  daily maintenance overhead.
- **Cognitive load:** System must surface what matters. I should not have to
  remember how it works.

**Tooling constraints:**
- Markdown + Git as the foundation.
- MkDocs for publication.
- AI tools (Claude, Cursor, Perplexity) are interchangeable; no vendor lock-in.

## Risk Profile and Rules

- Willing to risk: learning, prototypes, reversible experiments, new tools.
- Not willing to risk: family stability, irreversible financial commitments,
  legal exposure, reputation.
- Prefer reversible decisions. When in doubt, over-document and pause.

**Hard rules:**
- No major decisions without written rationale (in DECISIONS.md or domain file).
- Governance required on all structural repo changes.
- Privacy audit before publishing any personal context file.
- Pause if cognitively depleted; no major decisions under fatigue.
- No legal decision without written rationale.
- No financial execution without analysis first.
